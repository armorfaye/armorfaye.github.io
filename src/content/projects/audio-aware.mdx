## Problem & motivation

Deaf and hearing-impaired people miss critical auditory cues — a car horn, a
bike bell, someone calling out. AudioAware fills a gap in assistive technology
by not just detecting that a sound exists, but telling the user *what* it is and
*where* it is coming from, in real time.

## What I built

A wearable system made of smart glasses (four microphones + a camera) and a
wristband display, connected through a control loop that localizes and
classifies sounds and shows the user the source object and its direction.

## Technical approach

The system has three components:

- **JerryNet — direction of arrival.** A custom CNN takes a phase matrix built
  from four concurrent microphones and predicts one of nine directions. Audio
  is amplified to exaggerate inter-microphone volume differences.
- **CLAP audio classifier.** A fine-tuned Contrastive Language-Audio Pretraining
  model identifies the exact sound class from audio alone, working zero-shot at
  inference time.
- **Multimodal integration.** YOLOv9 generates object bounding boxes, and an
  audio-visual localization model (SIRA-SSL) selects the box that best matches
  the sound using a complete Intersection-over-Union (cIoU) criterion.

A FastAPI server coordinates the models, and the ESP32-S3-based hardware streams
audio/video and renders direction + object text/imagery on the wristband.

## Data collection

Because no suitable benchmark existed, I collected a custom dataset of 450 sets
from four concurrent microphones — 50 samples for each of nine directions —
split across indoor and outdoor environments to build robustness to background
noise.

## What's next

Collect a larger and more diverse dataset, optimize each component to cut
latency and server dependence, and explore reinforcement learning to adapt
sound-priority to user preference over time.
